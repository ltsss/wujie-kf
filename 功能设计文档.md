# 無界茶台自建客服系统 - 功能设计文档

## 一、项目概述

### 1.1 项目背景
- 现有 Tidio 客服系统功能受限，无法深度定制
- 需要集成 Dify AI 实现智能客服
- 需要支持"AI客服 + 人工客服"无缝切换

### 1.2 项目目标
- 打造专属客服系统，完全可控
- 集成 Dify AI，实现智能回复
- 支持转人工，提升用户体验
- 支持多客服协作

### 1.3 技术选型
| 模块 | 技术 |
|------|------|
| 后端 | Node.js + Express |
| 前端 | Vue.js 3 + Element Plus |
| 数据库 | MySQL |
| 实时通信 | Socket.io |
| AI 对接 | Dify API |

## 三、系统流程

### 3.1 访客接入流程
```
访客访问网站
    ↓
显示悬浮客服按钮
    ↓
访客点击打开聊天窗口
    ↓
显示欢迎语 + 常见问题
    ↓
访客发送消息
    ↓
AI 自动回复（Dify）
    ↓
判断是否需要转人工
    ↓
是 → 分配给在线客服
否 → 继续 AI 对话
```

### 3.2 转人工流程
```
访客点击"转人工"
    ↓
系统查找在线客服
    ↓
有客服在线 → 分配给客服
无客服在线 → 提示留言
    ↓
客服接入会话
    ↓
客服与访客对话
    ↓
客服结束会话
    ↓
访客评价
```

### 3.3 消息流转
```
访客发送消息
    ↓
服务器接收
    ↓
判断会话状态
    ↓
AI 模式 → 调用 Dify API
人工模式 → 推送给客服
    ↓
返回回复给访客
```

## 四、数据库设计

### 4.1 会话表 (conversations)
```sql
CREATE TABLE conversations (
  id VARCHAR(50) PRIMARY KEY,
  visitor_id VARCHAR(100) NOT NULL,
  visitor_name VARCHAR(100),
  agent_id VARCHAR(50),
  status ENUM('ai', 'waiting', 'active', 'closed') DEFAULT 'ai',
  source VARCHAR(50),
  ip VARCHAR(50),
  user_agent TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  closed_at TIMESTAMP NULL,
  INDEX idx_status (status),
  INDEX idx_agent_id (agent_id),
  INDEX idx_created_at (created_at)
);
```

### 4.2 消息表 (messages)
```sql
CREATE TABLE messages (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  conversation_id VARCHAR(50) NOT NULL,
  sender_type ENUM('visitor', 'agent', 'system', 'ai') NOT NULL,
  sender_id VARCHAR(100) NOT NULL,
  content TEXT NOT NULL,
  msg_type ENUM('text', 'image', 'file') DEFAULT 'text',
  is_read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_conversation_id (conversation_id),
  INDEX idx_created_at (created_at)
);
```

### 4.3 客服表 (agents)
```sql
CREATE TABLE agents (
  id VARCHAR(50) PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(255) NOT NULL,
  name VARCHAR(100) NOT NULL,
  avatar VARCHAR(255),
  status ENUM('online', 'offline', 'busy') DEFAULT 'offline',
  max_sessions INT DEFAULT 5,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_login_at TIMESTAMP NULL
);
```

### 4.4 快捷回复表 (quick_replies)
```sql
CREATE TABLE quick_replies (
  id INT AUTO_INCREMENT PRIMARY KEY,
  agent_id VARCHAR(50),
  title VARCHAR(100) NOT NULL,
  content TEXT NOT NULL,
  category VARCHAR(50),
  is_global BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 五、接口设计

### 5.1 访客接口

#### 创建会话
```http
POST /api/visitor/session/create
Request: { visitorId, source, url }
Response: { sessionId, welcomeMessage }
```

#### 发送消息
```http
POST /api/visitor/message/send
Request: { sessionId, content, type }
Response: { messageId }
```

#### 获取消息
```http
GET /api/visitor/messages?sessionId=xxx&lastId=xxx
Response: { messages: [] }
```

#### 转人工
```http
POST /api/visitor/transfer
Request: { sessionId }
Response: { status, queuePosition }
```

### 5.2 客服接口

#### 客服登录
```http
POST /api/agent/login
Request: { username, password }
Response: { token, agentInfo }
```

#### 获取会话列表
```http
GET /api/agent/sessions?status=active
Response: { sessions: [] }
```

#### 接入会话
```http
POST /api/agent/session/accept
Request: { sessionId }
Response: { success }
```

#### 发送消息
```http
POST /api/agent/message/send
Request: { sessionId, content, type }
Response: { messageId }
```

#### 结束会话
```http
POST /api/agent/session/close
Request: { sessionId }
Response: { success }
```

### 5.3 管理接口

#### 添加客服
```http
POST /api/admin/agent/create
Request: { username, password, name }
Response: { agentId }
```

#### 获取统计数据
```http
GET /api/admin/statistics?startDate=xxx&endDate=xxx
Response: { ... }
```
